Définition

Un framework est une plateforme logicielle préconstruite qui fournit des outils et des bibliothèques pour faciliter le développement de logiciels et accélérer le processus de programmation. Il permet aux développeurs de se concentrer sur les aspects uniques de leur application plutôt que de réinventer la roue pour des fonctions de base. Les frameworks sont utilisés dans de nombreux langages de programmation tels que JS, php, Python, Java et bien d'autres, et offrent une architecture de base pour les applications web, les applications mobiles, les logiciels de bureau et d'autres types de logiciels.

Mon expérience

J'ai eu à faire à plusieurs framework depuis mon BTS, la première fois était pour le projet Tryat. Ce projet serait la base de notre oral final, il nous a fallu le développer rapidement pour avoir assez vite du concret et pouvoir tester toutes les connexions via l' API de la balise SIGFOX. Nous avions dû utiliser un framework assez connu comme le "père" de CMS (Content Management System) très répandu de nos jours comme Drupal ou encore Wordpress. Ce framework c'est Symfony. L'aspect Modèle-Vue-Controleur a permis une prise en main très rapide, un de mes collègue s'était occupé de la partie front (Vue) du site et de mon côté la mise en place du backend (Modèle, Controleur).

Puis j'ai eu l'occasion de travailler avec 2 framework en même temps sur le projet Thalys. Un premier framework js, React. C'est une technologie avec le vent en poupe. L'avantage de ce framework est la possibilité de n'avoir qu'une seule page dont le contenu change, cela évite d'avoir des redirections ou chargement de page qui peuvent nuire à l'expérience utilisateur si trop long. La partie compte utilisateur du site web Thalys est complétement en React. Sa prise en main peut paraitre difficile mais ils proposent un tuto très bien fait pour pouvoir comprendre les mécanismes de ce dernier.
L'autre framework est un enfant de Symfony, Drupal. C'est donc un CMS, il permet de proposer un back-office (à ne pas confondre avec le backend) permettant d'ajouter du contenu à un site web. Exemple si vous souhaité créer une nouvelle page avec dedans un titre et un formulaire (inscription, newsletter, etc..), il vous suffit simplement de le créer depuis le back-office. Cela donne une grande autonomie aux clients et de pouvoir moduler leur site web à leur guise. L'avantage de Drupal, c'est que ce back-office est complétement modulable étant donné que les formulaires sont en fait des tableaux (array) ! Ce qui nous permet de les modifier comme bon nous semble.

Ensuite ?

Dans le contexte professionnel actuel, les frameworks sont devenus un élément clé du développement logiciel. Ils sont utilisés pour développer rapidement des applications de haute qualité, avec une structure solide et un code maintenable. Les frameworks sont particulièrement utiles pour les entreprises qui cherchent à développer des applications de manière agile, en réduisant les délais de commercialisation et en optimisant les processus de développement. Les frameworks sont également de plus en plus utilisés dans les technologies émergentes telles que l'Internet des objets (IoT) et l'intelligence artificielle (IA), où ils permettent aux développeurs de travailler sur des projets plus complexes en se concentrant sur des fonctionnalités spécifiques plutôt que de se préoccuper des détails techniques. Les frameworks sont donc une compétence clé pour les développeurs et les ingénieurs logiciels qui souhaitent travailler efficacement dans l'industrie du développement logiciel.
