Définition

Une API, Application Programming Interface, est une interface qui permet de connecter un service, comme un site web par exemple, à un autre service pour qu'ils puissent s'échanger des données ou fonctionnalités. Les API sont très utilisés, car elles permettent d'accéder à des services proposant des fonctionnalités prêtes à l'emploi ce qui évite de perdre du temps à réinventer la roue. Ou bien seulement récupérer des données pour les exploiter d'une quelconque manière que ce soit.

Mon expérience

Certains des projets sur lesquels j'ai pu travailler, ou travaille encore dessus actuellement, utilisent des API.

Un des projets actuellement en cours, le site de Thalys utilise des API pour communiquer avec des services tierss de l'entreprise Thalys afin de récupérer les informations concernant le client qui veut se connecter à son compte, la vérification est faites par un autre service. Ou encore pour que l'utilisateur puisse voir ses billets, qu'ils soient en cours, passées ou à venir. Avec toutes les informations relatives à ces derniers. Pour réserver un billet aussi, le formulaire sur la page d'accueil va envoyer les informations rentrées dedans à un service tiers pour nous afficher les billets disponible à l'achat.

Thermassist aussi utilise ce système d'API pour récupérer les informations des cures à venir pour les curistes. Ces API permettent de récupérer l'emploi du temps auprès d'entreprise qui s'occupe de fournir ces plannings. Une fois récupérées, les données sont exploitées pour être affichées dans un calendrier intégré dans le site que les curistes peuvent voir rapidement.

Je peux aussi vous parler rapidement de Tryat qui avait comme technologie centrale les API. Il fallait appeler une API fournie par un service tiers pour récupérer les données des balises. Une requête était envoyée avec dedans les informations de la balise comme son ID pour l'identifier. Et la réponse du service nous a permit d'analyser toutes les informations qu'avait pu collecter la balise.

Ensuite ?

Chaque projet sur lesquelles j'ai pu travailler jusqu'à aujourd'hui avaient une API. La manière dont ces API étaient appelés différé entre chacun de ces projets, ce qui ma permit de voir différente manière des les utiliser et de les mettre en place, et il m'en reste bien plus à découvrir. Ce qui reste le plus diifficile est de les prendre en mains et de comprendre où elles sont utilisé et ce qu'elles permettent de récupérer. Mais un peit peu de débogage et cette difficulté s'envol. Il me reste pas mal de chose à apprendre de ce côté là malgré le fait que je puisse mettre une API en place de manière autonome.

Les API sont omniprésent dans le monde du développpement. Il est certain que cela dure encore trés longtemps, tellement longtemps que je vais les utiliser jusqu'à ce que je m'arrête d'exercer. Il reste plus qu'à attendre et voir comment ces interfaces vont évoluer dans le futur. Elles vont sans doute devenir plus rapide encore, plus simple d'utilisation. Peu importe comment elles évolue, elles vont rester dans le paysage du développement en étant ses pilier. Il me parait primordiale pour quiconque débutant dans le développement, de jeter un oeil à ces dernières.
